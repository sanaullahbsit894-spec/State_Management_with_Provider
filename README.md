## ✨ Features

### Core Functionality

- ✅ **Create, Read, Update, Delete** tasks with full metadata
- 🎯 **Priority levels** — Low, Medium, High with color-coded indicators
- 🗂️ **Categories** — Personal, Work, Shopping, Health, Other
- 📅 **Due dates** with overdue detection and visual alerts
- 🔄 **Swipe gestures** — swipe right to complete, swipe left to delete
- 💾 **Local persistence** — tasks saved via SharedPreferences, survive app restarts

### State Management

- ⚡ **Provider** for reactive, app-wide state management
- 🔁 **Real-time UI updates** — no manual setState calls in screens
- 🧠 **Smart filtering** — filter by status (All / Active / Completed) and priority
- 📊 **Live stats** — total, completed, pending counts and animated progress bar

### UI / UX

- 🎨 **Custom dark theme** built on a handpicked forest-green palette
- 🌊 **Smooth animations** — staggered card entry, FAB scale, page transitions
- 📲 **Animated progress bar** tracking completion rate
- 🃏 **Slidable task cards** with haptic-style action panels
- 🌑 **Splash screen** with elastic logo animation

### Bonus — Push Notifications (FCM)

- 🔔 **Firebase Cloud Messaging** integration for push notifications
- 📬 **Foreground notification handling** via flutter_local_notifications
- 🛰️ **Background message handler** registered at app entry point
- 🔑 **Permission request flow** for iOS and Android 13+

## 🚀 Getting Started

### Prerequisites

- Flutter SDK `>=3.0.0`
- Dart SDK `>=3.0.0`
- Android Studio or VS Code with Flutter extension
- A Firebase project (for push notifications)

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/taskflow.git
cd taskflow
```

### 2. Install Dependencies

```bash
flutter pub get
```

### 3. Firebase Setup (for Push Notifications)

Install the FlutterFire CLI and configure your Firebase project:

```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Log in to Firebase
firebase login

# Auto-configure Firebase for this project
# (Generates lib/firebase_options.dart automatically)
flutterfire configure
```

> **Note:** Select your Firebase project and enable Android/iOS platforms. This generates `lib/firebase_options.dart` — do **not** create this file manually.

### 4. Android Configuration

In `android/app/src/main/AndroidManifest.xml`, make sure the following permissions and service are included inside `<manifest>`:

```xml
<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.POST_NOTIFICATIONS"/>
```

And inside `<application>`:

```xml
<service
    android:name="com.google.firebase.messaging.FirebaseMessagingService"
    android:exported="false">
    <intent-filter>
        <action android:name="com.google.firebase.MESSAGING_EVENT"/>
    </intent-filter>
</service>
```

### 5. Run the App

```bash
flutter run
```

## 🔄 State Management Architecture

TaskFlow uses the **Provider** package with a single `ChangeNotifier` class (`TaskProvider`) that owns all task state.

```
UI Widgets
   │
   ▼
context.read<TaskProvider>()     ← triggers actions (add, delete, toggle)
context.watch<TaskProvider>()    ← or Consumer<TaskProvider> for reactive UI
   │
   ▼
TaskProvider (ChangeNotifier)
   │  notifyListeners() on every change
   ▼
SharedPreferences                ← persists state between sessions
```

### Key Provider Methods

| Method                        | Description                               |
| ----------------------------- | ----------------------------------------- |
| `addTask(...)`                | Creates a new task and persists it        |
| `updateTask(task)`            | Updates an existing task by ID            |
| `deleteTask(id)`              | Removes a task permanently                |
| `toggleTaskCompletion(id)`    | Flips the completed state                 |
| `deleteCompleted()`           | Batch removes all completed tasks         |
| `setFilter(string)`           | Switches between All / Active / Completed |
| `setPriorityFilter(priority)` | Filters list by priority level            |

## 🔔 Push Notifications (FCM)

TaskFlow integrates Firebase Cloud Messaging to deliver real-time push notifications.

### How It Works

| Scenario              | Handler                                                       |
| --------------------- | ------------------------------------------------------------- |
| App in **foreground** | `FirebaseMessaging.onMessage` → shows local notification      |
| App in **background** | `FirebaseMessaging.onBackgroundMessage` → handled by Firebase |
| App **terminated**    | Firebase delivers notification via system tray                |

### Sending a Test Notification

1. Go to your [Firebase Console](https://console.firebase.google.com/)
2. Navigate to **Engage → Messaging → New Campaign → Notifications**
3. Set a title and body, choose your app, and click **Send test message**
4. Paste your device's FCM token (printed to console on first run)

## 📐 Architecture Decisions

| Decision                      | Reason                                                                                     |
| ----------------------------- | ------------------------------------------------------------------------------------------ |
| **Provider over setState**    | Eliminates prop drilling; centralizes all task logic in one place                          |
| **ChangeNotifier**            | Lightweight for this scope; Riverpod/Bloc would add overhead without benefit at this scale |
| **SharedPreferences**         | No server required; suitable for a single-user local task app                              |
| **SliverAppBar + SliverList** | Collapsible header with smooth scroll performance                                          |
| **flutter_slidable**          | Clean swipe-to-action UX without custom gesture detectors                                  |

## 🧩 Task Model

```dart
class Task {
  final String id;         // UUID v4
  String title;
  String description;
  bool isCompleted;
  Priority priority;       // low | medium | high
  TaskCategory category;   // personal | work | shopping | health | other
  DateTime createdAt;
  DateTime? dueDate;       // optional, triggers overdue detection
}
```

Tasks are serialized to JSON and stored as a `List<String>` in SharedPreferences.

## 🔭 Bonus: Riverpod / Bloc Comparison

| Feature        | Provider          | Riverpod          | Bloc                       |
| -------------- | ----------------- | ----------------- | -------------------------- |
| Learning curve | Low               | Medium            | High                       |
| Boilerplate    | Minimal           | Low               | High                       |
| Testability    | Good              | Excellent         | Excellent                  |
| Dev tools      | Basic             | Full              | Full                       |
| Best for       | Small–medium apps | Medium–large apps | Enterprise / complex flows |

For this scale of app, **Provider** is the right tool. A production app with complex async flows and multiple feature teams would benefit from **Riverpod** (code generation) or **Bloc** (strict separation of events and states).
